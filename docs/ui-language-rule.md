# UI Language Rule

Allen Stock Dashboard 的主要使用語言是台灣繁體中文（`zh-TW`）。介面可以保留必要的市場縮寫與產品名稱，但不可在同一層級任意切換中英文語氣。

## 基本規則

1. 導覽、頁面標題、按鈕、狀態、錯誤與操作提示使用繁體中文。
2. 股票代號、NASDAQ、SOX、ETF、MACD、API、V8.5 Pro+ 等通用縮寫可保留英文。
3. 品牌、provider 與技術產品名沿用官方拼法，例如 Yahoo Finance、Goodinfo、Supabase。
4. 英文 eyebrow 只作輔助分類，不得承載中文標題沒有表達的關鍵資訊。
5. 同一概念只用一個名稱；程式識別字可用英文，不應直接成為 UI 文案。

## 固定詞彙

| 使用 | 不使用 |
| --- | --- |
| 總覽 | Dashboard（單獨作為導覽文字） |
| 持股戰情中心 | Portfolio command |
| 風報比中心 | Risk / reward center |
| 主升段候選池 | Breakout pool（單獨呈現） |
| 今日禁碰股 | Avoid list |
| 市場燈號 | Market signal |
| 資料更新時間 | Last updated |
| 資料暫時無法取得 | Error / N/A |
| 觀察 | Watch |
| 減碼 | Reduce |

模型內部值應在 UI 層轉換：`strong_buy` →「積極關注」、`buy` →「偏多」、`watch` →「觀察」、`reduce` →「減碼」、`avoid` →「避開」。不得直接顯示 snake_case。

## 數字與時間

- 台股價格依商品最小跳動單位呈現；百分比最多兩位小數並顯示正負號。
- 成交量、金額可使用「萬、億」，tooltip 或詳情頁保留完整數值。
- 時區固定顯示 `Asia/Taipei`；必要時標示「台北時間」。
- 絕對時間使用 `YYYY/MM/DD HH:mm`，今日的高頻狀態可只顯示 `HH:mm:ss`。
- 缺值顯示「—」，不可用 `0` 假裝有資料。
- 過期資料需同時顯示最後更新時間與「資料已過期」，不可只改顏色。

## 投資語氣

- 陳述資料與模型判斷，不使用保證獲利、穩賺、必漲、必跌等承諾語。
- 建議文字需包含條件或理由，例如「量價結構轉強，可列入觀察」，避免只寫「買進」。
- 模型分數旁必須能辨識資料日期；Pro+ 詳情應提供信心度與資料完整度。
- 頁尾固定保留「資料僅供研究參考，不構成投資建議」。

## 狀態與錯誤

錯誤訊息採「發生什麼事＋使用者可以怎麼做」：

- 正確：「Yahoo Finance 暫時無法取得行情，畫面正顯示 09:30 的最後成功資料。」
- 不正確：「UPSTREAM_UNAVAILABLE」或「Something went wrong」。

載入、空資料、部分成功、過期與完全失敗必須是不同狀態。顏色不得是唯一線索，需搭配文字或圖示。

## 寫作檢查

新增 UI 文案前確認：用詞是否在固定詞彙表、時間與單位是否明確、英文是否真的必要、錯誤是否能採取行動、是否誤把模型訊號寫成投資保證。
